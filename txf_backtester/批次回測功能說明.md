# MTX 回測平台｜批次回測功能說明（適用 v0.6.6）

本文件用途：交給 AI（或協作者）時，讓對方不看程式碼就能正確使用批次回測。

## 1. 平台概觀

- 商品：MTX 小型臺指期（日K），資料為期交所每日行情建立的近月連續契約（prepared 檔：`data/prepared/MTX_stable_rollover_daily.csv`，2015-01 ～ 2026-06）。
- 回測核心規則：第 i 根收盤訊號成立 → 第 i+1 根開盤價進場（含滑價），單一持倉、不加碼、不反手，無未來函數。
- 出場優先順序（同一根K棒內）：斷頭開盤跳空 > 固定停損 > 固定停利 > 移動停損 > 斷頭盤中觸價 > 吊燈出場 > MACD反向 > 條件出場。吊燈與MACD反向為收盤確認、收盤價出場；停損停利為盤中觸價。
- 成本：手續費每邊 20 元、滑價每邊 1 點、期交稅每邊成交金額×0.00002（**v0.4.6 起預設計入**）。
- 斷頭模擬：反向浮動損失吃光安全緩衝金額（近250日最高價×25%×50元）即強制平倉，比較表會顯示斷頭次數。

## 2. 執行方式

兩個入口功能相同（上限20組、sweep、新比較欄位、年度分解都支援）：

- **網頁介面**：`streamlit run app.py` → 側欄「批次策略回測」→ 從策略投放箱選 JSON 或手動上傳。期交稅勾選預設開啟。
- **命令列**（適合 AI 自動執行）：

```bash
python run_batch_backtest.py                          # 讀取 _策略投放箱 最新的 JSON
python run_batch_backtest.py --batch 路徑/批次.json    # 指定 JSON
python run_batch_backtest.py --start 2015-01-01 --end 2023-12-31   # 指定回測區間（樣本內/樣本外用）
```

主要參數：

| 參數 | 預設 | 說明 |
|---|---|---|
| `--batch` | 投放箱最新 JSON | 批次策略 JSON 路徑 |
| `--record-dir` | 自動偵測 MTX Test Record | 結果輸出資料夾（Obsidian/Google Drive） |
| `--start` / `--end` | 資料起迄 | 回測區間，做樣本內外切分必用 |
| `--max-strategies` | 20 | 單批策略數上限 |
| `--no-tax` | 關 | 加上此旗標則不計期交稅 |
| `--fee` / `--slippage` | 20 / 1.0 | 成本敏感度測試時調整 |
| `--initial-capital` | 159000 | 報酬率分母 |

## 3. 批次策略 JSON 格式

### 3.1 一般模式（逐組手寫）

```json
{
  "batch_name": "batch_005_主題名稱",
  "strategies": [
    { "name": "01_策略名", "config": { ...完整策略config... } }
  ]
}
```

策略 config 結構（重點欄位）：

```json
{
  "symbol": "MTX",
  "timeframe": "1D",
  "direction": "both",
  "entry_long":  { "logic": "AND", "conditions": [ {"type": "ma_bullish_alignment", "ma_type": "SMA", "periods": [20, 60, 120]}, {"type": "macd_hist_cross_up", "fast": 12, "slow": 26, "signal": 9} ] },
  "entry_short": { "logic": "AND", "conditions": [ {"type": "ma_bearish_alignment", "ma_type": "SMA", "periods": [20, 60, 120]}, {"type": "macd_hist_cross_down", "fast": 12, "slow": 26, "signal": 9} ] },
  "exit": {
    "use_chandelier": true, "chandelier_period": 22, "chandelier_mult": 2.5,
    "use_macd_reverse": true,
    "use_fixed_stop": true, "stop_points": 200,
    "use_take_profit": false, "take_profit_points": 500,
    "use_trailing_stop": false, "trailing_points": 150
  }
}
```

進場條件積木（`conditions[].type`）常用清單：`close_above_ma`／`close_below_ma`、`ma_cross_up`／`ma_cross_down`、`ma_bullish_alignment`／`ma_bearish_alignment`（多空排列，periods 陣列）、`ma_slope_up`／`ma_slope_down`、`macd_hist_cross_up`／`macd_hist_cross_down`（MACD翻多/翻空）、`macd_hist_positive`／`macd_hist_negative`、`macd_dif_above_zero`／`macd_dif_below_zero`、布林通道系列（`close_above_bollinger_mid`、`close_cross_up_bollinger_lower` 等）、KD 系列。完整清單見 `condition_blocks.py` 的 `@register` 名稱。`logic` 支援 AND / OR。

### 3.2 sweep 參數掃描模式（v0.4.6 新增，建議優先使用）

只掃一個參數時，不必手寫 N 組，給 base + 值陣列即可：

```json
{
  "batch_name": "batch_005_吊燈倍數掃描",
  "strategies": [
    { "name": "01_對照_舊高績效基準", "config": { ... } },
    { "name": "02_對照_舊低回撤基準", "config": { ... } }
  ],
  "sweep": {
    "base": { ...完整策略config（不含name）... },
    "param": "exit.chandelier_mult",
    "values": [2.0, 2.25, 2.5, 2.75, 3.0, 3.25],
    "name_prefix": "MA20_60_120_吊燈22x"
  }
}
```

- `param` 用點記法指到任何巢狀欄位，例如 `exit.stop_points`、`exit.chandelier_period`。
- `strategies`（固定對照組）與 `sweep` 可並存，總數 ≤ 上限。
- `sweep` 可以是陣列（多個掃描共用一批）。
- 展開後策略名 = `name_prefix` + 值。

限制：sweep 對「值是陣列」的參數（如均線 periods）不適用點記法逐值展開，均線組合請用一般模式逐組寫。

## 4. 輸出結構

每次執行會在 record-dir 建立 `MTX_批次回測_<時間戳>_<batch_name>/`：

```
00_批次回測總覽.md          ← 批次比較表（Obsidian 可讀）+ 檢討欄位
batch_comparison.csv        ← 比較表 CSV
<批次名>.zip                ← 整批打包
01_<策略名>/
  00_策略回測摘要.md        ← 單策略摘要 + 年度分解表
  trades.csv                ← 交易明細（中文欄位：訊號日/進場日/出場日/方向/出場原因/損益金額...）
  metrics.csv               ← 全部績效指標
  equity_curve.csv          ← 資金曲線（日期/資金/回撤）
  yearly_stats.csv          ← 年度分解（v0.4.6 新增）
  strategy_config.json      ← 該組完整設定（可直接複製當下一批的 base）
```

比較表欄位：總損益(元)、總報酬率(%)、年化報酬率(%)、最大回撤(元)、**報酬回撤比**、獲利因子、期望值(元/筆)、交易次數、勝率(%)、最大連續虧損(次)、平均持倉K棒數、資金持續未創新高交易天數、斷頭資訊。

指標解讀慣例：
- **報酬回撤比**＝總損益÷|最大回撤|，為主要排序指標（同時反映「績效好」與「回撤低」）。
- 最大回撤以權益曲線（已實現+未實現、收盤基準）計算，盤中回撤可能略被低估。
- 報酬率分母為初始資金（預設 159,000）。

## 5. 兩策略明細比對工具

```bash
python compare_trades.py "批次資料夾/04_策略A" "批次資料夾/03_策略B"
python compare_trades.py a/trades.csv b/trades.csv --out 輸出資料夾
```

輸出 `交易比對_A_vs_B.md`：A/B 總損益差、相同進場相同出場筆數、**相同進場不同出場明細**、只有單方有的交易、逐年損益差。

用途判準：若兩組參數的差異只來自少數幾筆「不同出場」交易（例如 11 年只差 4 筆），視為雜訊等級差異，不應據此細調參數。

## 6. 給 AI 的操作守則

1. 每批只測一個主題（一個參數或一個結構），其餘固定。
2. 每批保留固定對照組（舊高績效基準、舊低回撤基準、目前主力候選）。
3. 排序先看報酬回撤比，再看總損益與回撤絕對值；勝率為次要參考。
4. 兩組差異小（總損益差 < 5% 或明細比對只差幾筆）→ 視為無差異，不得宣告新最佳值。
5. 調參一律用 `--start/--end` 限定樣本內區間；樣本外區間留給最終驗證（見測試計畫架構 v2）。
6. 執行後把 `batch_comparison.csv` 與有疑問策略的 `yearly_stats.csv`、明細比對報告一起讀，再下結論。


## ATR 標準化分段出場 JSON 欄位

```json
{
  "use_profit_tier_chandelier": true,
  "profit_tier_threshold_mode": "entry_atr",
  "profit_tier_reference": "max_favorable",
  "profit_tier_atr_multiples": [2, 4, 8],
  "profit_tier_mults": [2.5, 3.0, 3.5, 5.0],
  "use_profit_scaled_macd_exclusion": true,
  "macd_reverse_exclude_atr_multiple": 4
}
```

`entry_atr` 固定取訊號根 ATR，不取進場當根 ATR。


## v0.7.0 通用量化研究平台

- 多數研究能力已改由策略 JSON 控制。
