# V0.4.6 CHANGELOG（2026-07-09）

目標：讓批次回測直接服務「找出績效較好、勝率較高、最大回撤較低的策略」。

## metrics.py

- 新增 `報酬回撤比`（總損益 ÷ |最大回撤|）：建議作為策略排序的主要指標。
- 新增 `最長回撤持續(K棒)`：權益創新高後回到新高的最長間隔。
- 新增 `yearly_stats(trades, equity)`：年度分解（每年損益／交易次數／勝率／年度內最大回撤，年度內回撤每年重設高點）。
- 舊指標鍵名全部保留，相容既有匯出。

## run_batch_backtest.py

- 批次比較表新增欄位：報酬回撤比、Profit Factor、期望值(元/筆)、最大連續虧損(次)、平均持倉K棒數、最長回撤持續(K棒)。
- 每策略資料夾自動輸出 `yearly_stats.csv`，策略摘要 md 內含年度分解表。
- 批次 JSON 新增 **sweep 參數掃描模式**：給 base 策略 + 參數路徑 + 值陣列，自動展開成多組策略（詳見批次回測功能說明）。
- 單批上限 10 → **20**，可用 `--max-strategies` 調整。
- **期交稅預設計入**（稅率取自 config.py `SYMBOLS["MTX"]["tax_rate"]`）；`--no-tax` 可關閉。舊 `--use-tax` 保留相容、已無作用。
  - 注意：因此 v0.4.6 起的總損益會略低於 v0.4.5（每邊約 20~25 元/口），屬正常。

## compare_trades.py（新檔案）

- 兩策略交易明細比對：以「訊號日+方向」對齊，分類為 相同進場相同出場／相同進場不同出場／只有A／只有B，並輸出逐年損益差異。
- 用途：驗證兩組參數的差異是來自結構性改變還是少數幾筆交易（雜訊）。
- 實測範例：MA20/60/120 吊燈 22x2.5 vs 22x3.0，11.5 年只有 4 筆交易不同、差異 9,100 元 → 屬雜訊等級。

## app.py（Streamlit 網頁介面）— v0.4.6b 補齊

第一版 v0.4.6 只改了命令列入口，網頁介面的批次回測是獨立程式碼，已同步修改：

- 批次上限 10 → 20（含按鈕文字）。
- `parse_strategy_batch` 支援 sweep 展開（與命令列相同 JSON 格式）。
- 批次比較表（畫面 dataframe、總覽 md、batch_comparison.csv）加入全部新欄位。
- 單策略輸出加入 `yearly_stats.csv` 與摘要內年度分解表。
- 「計入期交稅」勾選預設改為開啟（可手動關閉）。
