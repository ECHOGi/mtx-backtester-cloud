# v0.3.3 操作流程重整版

回測核心零修改（backtester.py / correctness.py / self_check_correctness.py /
data_loader.py / continuous_contract.py 未動）。
strategies.py 僅「增量」新增 evaluate_entry()：支援多個進場組合
（組合內 AND、組合之間 OR），單一組合行為與舊版完全相同。
self_check_correctness.py 仍 PASS 8 cases。

## 修改內容

1. 策略設定面板（st.dialog 彈出視窗 + st.form）
   - 側欄與主畫面都有「🎮 策略設定面板」按鈕。
   - 流程：1️⃣ 勾進場條件 → 2️⃣ 勾出場條件 → 3️⃣ 按「▶ 開始回測」。
   - 「▶ 開始回測」旁有「取消」按鈕；取消不套用任何變更。
2. 修正「每調一個欄位就重跑回測」
   - 所有策略設定包在 form 內，提交前不會觸發任何計算。
   - 回測結果存於 session_state，只有按「開始回測」才重新計算。
   - 側欄成本/資料設定變更後，畫面顯示「⚠ 設定已變更」提示，不自動重跑。
   - 圖表顯示設定與顯示範圍只重畫圖，不重新回測。
3. 進出場條件遊戲化（條件組合積木）
   - 多單/空單各 2 個進場組合（A/B）：組合內 AND、組合間 OR。
   - 每個條件都有中文名稱＋簡短說明；新增 KD（黃金/死亡交叉、高低檔）
     與 RSI（強勢/弱勢）、均線交叉、均線排列、量能放大等 9 種條件。
   - 交易明細「進場條件」會標註觸發組合（例：組合1：MACD翻多 且 …）。
4. 左側欄重新定位（低頻設定）：策略設定面板／操作模式／圖表顯示／
   交易成本與資金／資料設定／參數檔。移除「選策略」下拉，
   主畫面固定顯示「目前內建可選用策略：MACD＋布林線＋吊燈出場＋KD＋RSI」。
5. 策略 JSON 新增 ui_combos 欄位，載回可完整還原遊戲化勾選。
