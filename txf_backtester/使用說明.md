# 台指期回測工具 使用說明（v0.6.6）

## 1. 這是什麼？

這個工具目前不是傳統 Windows EXE 軟體，而是：

```text
Python 專案 + Streamlit 本機網頁介面
```

啟動後會在瀏覽器開啟：

```text
http://localhost:8501
```

你是在瀏覽器裡操作回測平台。

---

## 2. 資料要放哪裡？

請在 `txf_backtester` 資料夾底下建立或確認有 `data` 資料夾。

資料放置方式如下：

```text
txf_backtester/
├─ app.py
├─ 啟動台指期回測工具.bat
├─ data/
│  ├─ 2015_fut.csv
│  ├─ 2016_fut.csv
│  ├─ 2017_fut.csv
│  ├─ 2018_fut.csv
│  ├─ 2019_fut.csv
│  ├─ 2020_fut.csv
│  ├─ 2021_fut.csv
│  ├─ 2022_fut.csv
│  ├─ 2023_fut.csv
│  ├─ 2024_fut.csv
│  └─ 2025_fut.csv
```

左側「資料與換倉進階」內的「CSV 資料夾路徑」預設是：

```text
data
```

如果資料已經放在上面的 `data` 資料夾，這個欄位不用改。

---

## 3. 最簡單啟動方式

直接雙擊：

```text
啟動台指期回測工具.bat
```

它會自動執行：

```text
py -m streamlit run app.py
```

第一次使用時，如果還沒安裝必要套件，啟動檔會先安裝 `requirements.txt` 裡的套件。

---

## 4. 使用期間不要關閉黑色視窗

雙擊啟動後，會出現一個黑色 CMD 視窗。

請注意：

```text
這個黑色視窗就是本機伺服器。
使用回測平台時不要關閉它。
```

如果把黑色視窗關掉，瀏覽器裡的回測平台也會停止。

用完後，直接關閉黑色視窗即可停止工具。

---

## 5. v0.6.6 介面使用重點

1. 左側上方會顯示版本編號，可用來確認重新部署是否成功。
2. 操作環境整合為「雲端・桌機／雲端・手機／本機・桌機」。
3. 常用批次回測置於左側前段；資料路徑、成本、圖表與其他來源收進進階區。
4. 欄位說明改用旁邊的「?」提示，需要時再點開。
5. 策略設定改為多單、空單、出場與進階參數四區，未啟用的出場參數不顯示。
6. 桌機與手機共用清新配色、圓角按鈕與響應式版面。
7. 任何設定變更都不會自動重跑，需按下執行按鈕。

---

## 6. 圖表顯示不影響回測結果

左側「圖表顯示」區只控制畫面要不要顯示某些線圖，例如：

```text
K 線
進場／出場標記
布林通道
吊燈線
均線
成交量
MACD 副圖
KD 副圖
權益曲線
```

這些勾選只影響畫面，不會改變：

```text
進場點
出場點
損益
勝率
最大回撤
交易明細
```

會影響回測結果的是：

```text
策略參數
出場條件
交易方向
成本設定
回測期間
商品與資料
```

---

## 7. Streamlit 右上角的 Deploy / 部署

如果仍看到右上角的 Deploy / 部署，不需要按。

那是 Streamlit 的開發工具列，不是本工具的功能。

---

## 8. 停止工具

用完後：

1. 關掉瀏覽器分頁。
2. 關掉黑色 CMD 視窗。

這樣回測平台就停止了。

## 雙擊啟動檔補充

若 `啟動台指期回測工具.bat` 雙擊後仍然異常，可改用英文檔名備用啟動檔：

```text
START_BACKTESTER.cmd
```

兩個檔案功能相同。新版啟動檔內部指令已改成英文與 ASCII，避免 Windows CMD 因中文編碼造成一閃而過。

若啟動失敗，黑色視窗會停住，不會立刻消失。請截圖錯誤訊息再回報。


## 6. ATR 標準化分段出場

在「策略設定 → 出場條件 → ATR 標準化分段出場」啟用。建議門檻先以 `2, 4, 8` ATR 測試，吊燈倍數可先用 `2.5, 3.0, 3.5, 5.0`。

「進場可用 ATR」採訊號根 ATR，也就是實際進場開盤前已知的最後一根完整 K 棒 ATR，避免未來函數。
